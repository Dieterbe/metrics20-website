Tools that currently support metrics 2.0

Statsdaemon

vimeo/statsdaemon is a statsd-like aggregator. If it detects input metrics to be in metrics 2.0 format, it will report the output in metrics 2.0 format too. So for example a metric with metric_type=count and unit=B will leave, after computing the rate, with tags metric_type=rate and unit=MB/s.

Graph-Explorer

Graph-Explorer is the prototypical metrics 2.0 dashboard implementation for graphite. It provides graphs, dashboards and alerting rules based on a input queries. It requires a database of metrics 2.0, which can be populated by carbon-tagger or structured_metrics (see below).

Carbon-tagger

Carbon-tagger is a service that you can put in your metrics pipeline, it automatically maintains a database of all metrics 2.0 it sees.

Structured Metrics

structured_metrics is a small python utility that comes with Graph-Explorer, it updates metrics 2.0 in a database by parsing legacy metrics. I.e. you can use this to keep using your old, existing plugins as metrics 2.0. It comes with plugins for diamond, collectd and openstack, it's easy to add your own.
